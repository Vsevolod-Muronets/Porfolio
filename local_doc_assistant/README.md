# Административный ассистент

Локальный RAG-ассистент для работы с документами. Все данные остаются на вашем компьютере - ничего не уходит в облако.

## Возможности

- **База знаний** - задавайте вопросы по нормативным документам организации (PDF, DOCX, TXT)
- **Редактор заданий** - загрузите DOCX с заданием для договора ГПХ, получите версию с юридически грамотными формулировками

## Стек

- [Ollama](https://ollama.com) - локальный запуск LLM
- [Qwen3:8b](https://ollama.com/library/qwen3) - языковая модель
- [Qwen3-embedding:4b](https://ollama.com/library/qwen3-embedding) - модель эмбеддингов
- [ChromaDB](https://www.trychroma.com) - векторная база данных
- [LangChain](https://python.langchain.com) - оркестрация RAG
- [Flask](https://flask.palletsprojects.com) - веб-интерфейс

## Требования

- Python 3.10+
- [Ollama](https://ollama.com/download)
- 16 GB RAM (рекомендуется)

## Установка и запуск

```bash
# Создать виртуальное окружение
python3 -m venv envi

# Первый запуск - установка зависимостей и скачивание моделей
bash setup.sh

# Последующие запуски
bash run.sh
```

Открыть в браузере: **http://localhost:7860**

## Структура проекта

```
private-rag/
├── local_doc_assistant.py   # основной скрипт
├── setup.sh           # первый запуск
├── run.sh             # обычный запуск
├── requirements.txt   # зависимости
├── norms/             # нормативные документы (PDF/DOCX/TXT)
├── uploads/           # временные файлы, а также отредактированные файлы с заданиями (создается автоматически)
└── chroma_db/         # векторная база (создается автоматически)
```

## Добавление документов в базу знаний

Положите PDF, DOCX или TXT файлы в папку `norms/` и перезапустите с флагом пересборки базы:

```bash
python3 gph_assistant.py --rebuild
```
